2. 需求背景与目标

2.1 现状问题与痛点

当前电子吧唧产品存在以下痛点：

  1. 内容消费场景单一：产品仅支持随硬件购买时赠送官方授权吧唧图片，缺乏持续的内容消费和付费场景
  2. 用户粘性不足：用户购买设备后，缺少持续参与的动力，产品生命周期价值（LTV）较低
  3. 商业变现路径有限：仅靠硬件销售盈利，缺少软件侧的增值服务收入
  4. 社交分享动力弱：用户缺少收集和展示稀有吧唧的成就感，不利于病毒式传播

2.2 需求提出的必要性

通过引入"抽选商城"功能，可以：

  • 为用户提供持续的内容获取渠道，增加产品使用频次
  • 通过盲盒抽选的趣味性，提升用户体验和付费意愿
  • 构建"收集-升级-展示"的完整用户闭环，增强产品粘性
  • 为运营提供灵活的营销活动工具（限量发售、概率调整等）

2.3 业务目标

3. 用户与使用场景

3.1 核心用户

  • IP 粉丝型用户：热爱特定动漫/游戏 IP，愿意为周边付费
  • 收藏爱好者：喜欢收集各类虚拟/实体周边，追求稀有度
  • 设备玩家：已购买电子吧唧设备，希望个性化展示
  • 价格敏感型用户：关注性价比，喜欢优惠活动

3.2 典型场景

  1. 用户进入 APP 浏览 IP 主题包列表，查看概率公示
  2. 用户选择 IP 包，选择单抽/10 连抽/整包购买方式进行抽选
  3. 用户消耗免费次数或付费进行抽选，观看动效并查看结果
  4. 用户进入吧唧图库管理已获得的图片，进行合并升级或导入设备
  5. 用户抽中 SSR/SR 级别吧唧后，填写收货地址等待实体配送

5. 详细方案

5.1 APP 端功能

5.1.1 IP 主题包列表页

功能描述：展示所有可抽选的 IP 主题包，用户可浏览并选择感兴趣的 IP 包进入详情页。

详细逻辑
  • 展示所有已上架的 IP 主题包卡片
  • 每个卡片包含：IP 封面图、IP 名称、包含图片数量、价格区间
  • 支持按"热门"、"新品"、"即将结束"排序
  • 点击卡片进入 IP 包详情页
  • 仅展示运营后台配置为"已上架"且当前时间在上下架时间范围内的 IP 包
  • "即将结束"标签：距离下架时间 < 24 小时的 IP 包
  • "新品"标签：上架时间 < 7 天的 IP 包
交互设计
  • 卡片式布局，每行展示 2 个 IP 包
  • 标签使用不同颜色区分（热门-红色、新品-蓝色、即将结束-橙色）
  • 支持下拉刷新和上拉加载更多
数据流向
前端加载页面 → 获取 IP 包列表数据 → 展示 IP 包卡片
→ 用户点击卡片 → 跳转 IP 包详情页

5.1.2 IP 包详情页

功能描述：展示 IP 包完整信息，包括概率公示和购买选项，用户可在此选择抽选方式。

详细逻辑
  • 展示 IP 包完整信息：封面大图、IP 名称、简介、包含图片数量
  • 概率公示表格（强制合规要求）：
等级说明抽取概率库存状态
SSR超稀有2%限量 XXX 张
SR稀有18%限量 XXX 张
R普通80%不限量
  • 购买选项区域：
    • 单抽：9.9 元/次，显示用户当前免费次数
    • 10 连抽：99 元/次，标注"必中 1 张 SR 及以上"
    • 整包购买：显示配置价格（如"50 元/包，含 10 张"）
  • 用户已收集进度：已拥有 X/总 Y 张
  • 优先消耗免费次数，免费次数用完后引导付费
  • 10 连抽必中等级由运营后台配置（SSR 或 SR）
  • 库存为 0 的等级，在概率表中显示"已售罄"，但概率重新分配到其他等级
交互设计
  • 概率公示表格必须清晰可见，不可折叠或隐藏
  • 购买选项使用卡片式布局，突出价格信息
  • 免费次数使用高亮标识，引导用户优先使用
数据流向
前端加载页面 → 获取 IP 包详情和概率配置 → 展示页面
→ 用户选择购买选项 → 进入对应抽选流程

5.1.3 单抽功能

功能描述：用户进行单次抽选，展示抽选动效后即时出结果。

详细逻辑
  • 展示抽选动效（类似扭蛋机/开盲盒动画，持续约 3-5 秒）
  • 动效结束后，展示获得的吧唧图片
  • 自动触发抽选结算弹窗
  • 根据 IP 包配置的各等级概率，随机抽取一张图片
  • 若抽到 SSR/SR 且库存 > 0，则扣除对应库存
  • 若库存为 0，则重新抽取（降级到下一等级）
  • 记录抽选流水（用户 ID、IP 包 ID、获得图片 ID、时间戳）
交互设计
  • 动效需有"悬念感"，在揭晓结果前有光影/粒子特效
  • SSR 级别图片需有特殊动效（如金色光芒、全屏特效）
  • 支持跳过动效（右上角"跳过"按钮）
数据流向
用户触发单抽 → 扣费或消耗免费次数 → 服务端随机抽取
→ 返回结果 → 展示动效 → 展示结算弹窗

5.1.4 10 连抽功能

功能描述：用户进行 10 次连续抽选，必中 1 张指定等级图片。

详细逻辑
  • 不展示抽选动效（区别于单抽），直接展示 10 张结果列表
  • 结果列表按等级降序排列（SSR > SR > R）
  • 自动触发抽选结算弹窗
  • 10 张中必须有 1 张达到运营配置的必中等级（SSR 或 SR）
  • 其余 9 张按正常概率抽取
  • 若必中等级库存为 0，则降级到下一等级
交互设计
  • 结果列表支持左右滑动查看
  • 突出显示必中的那张图片（标注"保底"）
  • 使用网格布局展示 10 张图片
数据流向
用户触发 10 连抽 → 扣费 99 元 → 服务端抽取 10 张图片
→ 必中逻辑处理 → 返回结果 → 展示结果列表 → 展示结算弹窗

5.1.5 整包购买功能

功能描述：用户购买整包图片，直接获得确定性图片集合。

详细逻辑
  • 不展示抽选动效
  • 付费成功后，直接跳转"我的吧唧图库"页面
  • 在图库页面顶部展示 Toast 提示："恭喜获得 X 张吧唧图片，已存入您的图库"
  • 整包购买的图片是确定性的（非随机），包含该 IP 包的所有图片（或配置的部分图片）
  • 扣除对应库存，记录订单流水
交互设计
  • 付费成功后展示加载动画
  • 跳转图库时展示平滑过渡动画
数据流向
用户触发整包购买 → 扣费 → 服务端发放整包图片
→ 跳转图库页面 → 展示 Toast 提示

5.1.6 抽选结算弹窗

功能描述：展示抽选结果，提供后续操作入口。

详细逻辑
  • 弹窗展示本次抽选获得的所有图片
  • 展示图片的等级标识（SSR/SR/R）
  • 若抽到 SSR/SR，展示"恭喜获得稀有吧唧！"庆祝文案
  • 提供两个操作按钮：
    • "去查看"：跳转"我的吧唧图库"页面
    • "继续抽选"：关闭弹窗，返回 IP 包详情页继续抽选
  • 弹窗不可通过点击外部区域关闭（强制用户做出选择）
  • 若抽到 SSR/SR 且用户未填写过收货地址，弹窗底部增加提示："稀有吧唧支持实体配送，请填写收货地址"，并提供"去填写"链接
交互设计
  • 弹窗支持左右滑动查看所有获得图片（10 连抽时）
  • SSR 图片展示时增加粒子动效
  • 按钮使用主次按钮区分（"去查看"为主按钮，"继续抽选"为次按钮）
数据流向
抽选完成 → 触发结算弹窗 → 用户选择操作
→ 跳转图库 或 返回详情页

5.1.7 我的吧唧图库

功能描述：用户管理已获得的吧唧图片，查看收集进度，进行合并升级或导入设备。

详细逻辑
  • 顶部展示用户设备连接状态（复用 1 期功能）
  • Tab 切换：
    • 已拥有：展示用户已获得的所有吧唧图片，按 IP 分组或时间倒序
    • 待解锁：展示用户尚未获得的该 IP 包内的图片（灰色遮罩）
  • 每张图片展示：吧唧图片、名称、等级标识
  • 点击已拥有图片：进入图片详情页，提供"导入设备"、"合并升级"、"分享"按钮
  • 点击待解锁图片：展示"去抽选"按钮，跳转对应 IP 包详情页
  • 已拥有图片按"获得时间"倒序排列
  • 支持按 IP 筛选和按等级筛选
交互设计
  • 图片卡片采用网格布局（每行 2 张）
  • 底部展示收集进度："已拥有 123/200 张"
  • 待解锁图片使用灰色遮罩和锁图标
数据流向
用户进入图库 → 加载已拥有/待解锁图片列表 → 展示图片卡片
→ 用户点击图片 → 进入详情页或抽选页

5.1.8 合并升级功能

功能描述：用户将多张低等级吧唧图片合并升级为高等级图片。

详细逻辑
  • 升级规则：
    • 规则 1：消耗 2 张相同吧唧图片 → 升级为高一级吧唧（随机从高一级的图片中抽取 1 张）
    • 规则 2：消耗 4 张不同吧唧图片（同一 IP 包内）→ 随机升级为高一级吧唧（从该 IP 包高一级的图片中抽取 1 张）
  • 升级前展示确认弹窗，列出消耗图片和可能获得的图片范围
  • 升级过程展示动效（类似抽选动效，但更简短）
  • 升级完成后展示结果弹窗
  • R 级可升级为 SR 级，SR 级可升级为 SSR 级，SSR 级不可升级
  • 升级不消耗库存（因为是用户已拥有的图片合成）
  • 升级后的图片记录新的获得时间
  • 被消耗的图片从"已拥有"中移除，但保留抽选记录
交互设计
  • 在图库页面，用户长按某张图片进入"多选模式"
  • 选择 2 张相同或 4 张不同图片后，底部出现"合并升级"按钮
  • 若选择的图片不符合规则，按钮置灰并提示原因
  • 升级动效展示材料消耗和新图片生成过程
数据流向
用户选择图片 → 触发合并升级 → 校验规则
→ 消耗材料图片 → 抽取高一级图片 → 添加新图片到图库

5.1.9 SSR/SR 实体吧唧申请邮寄

功能描述：SSR/SR 实体吧唧申请邮寄

详细逻辑
  • 用户选择未邮寄的实体吧唧项目，选择收件人地址后二次确认，递交生成0元订单，成功后提示用户在我的订单里查看物流信息。
  • 选择地址、新建地址、订单后续处理为已有功能。
交互设计
  • 使用表单形式，必填项使用红色星号标识
  • 提交按钮固定在底部
数据流向
用户中奖 SSR/SR → 用户选择未邮寄的实体吧唧项目 → 用户选择收件人地址 → 用户提交表单
→ 服务端保存地址 → 展示成功提示

5.2 运营后台端功能

5.2.1 IP 包管理

功能描述：运营人员在后台配置和管理 IP 主题包的上下架、库存、概率等信息。

详细逻辑
  • IP 包列表：展示所有 IP 包，支持按状态（上架/下架）筛选
  • 新增/编辑 IP 包：
    • 基本信息：IP 名称、封面图、简介
    • 上下架时间：开始时间、结束时间
    • 图片库管理：上传该 IP 包包含的所有吧唧图片，设置每张图片的等级（SSR/SR/R）
    • 库存配置：为每个等级设置库存总量（R 级可设置为"不限量"）
    • 概率配置：设置各等级的抽取概率（总和必须为 100%）
    • 10 连抽必中等级：选择 SSR 或 SR
    • 整包配置：整包包含的图片列表、整包价格
  • 上下架操作：手动上下架 IP 包，或到达配置时间自动上下架
  • 概率配置必须进行总和校验（SSR% + SR% + R% = 100%）
  • 库存扣减实时同步，库存为 0 时前端自动显示"已售罄"
  • 已上架的 IP 包，修改概率和库存需二次确认

  • 概率配置使用输入框 + 实时校验提示
  • 图片上传支持拖拽和批量上传
数据流向

7. 异常与边界处理

  • 网络异常：抽选过程中网络中断，保留抽选请求，网络恢复后展示结果；若支付成功但未收到结果，自动重试或退款
  • 库存异常：抽选瞬间库存为 0，自动降级到下一等级，弹窗提示"该等级已售罄，已为您升级抽取"
  • 支付异常：支付成功但未发放图片，后台定时任务核对订单与发放记录，自动补发；支付失败提示"支付失败，请重试"，不扣除免费次数
  • 升级异常：升级过程中网络中断，升级操作为原子事务，要么成功要么失败；失败时不消耗材料图片
  • 高一级图片库存为 0：升级失败，提示"该等级暂无可升级图片，请稍后再试"，不消耗材料
  • 空状态处理：
    • IP 包列表页（无上架 IP）：展示"暂无可抽选的 IP 包，敬请期待"
    • 我的吧唧图库（未获得图片）：展示"您还没有吧唧图片，快去抽选吧！"
    • 待解锁 Tab（已全部解锁）：展示"恭喜！您已集全该 IP 包所有图片"
  • 权限限制：未登录用户点击抽选，跳转登录页，登录后返回原页面；未绑定设备用户允许抽选和收集，但导入设备时引导绑定设备
  • 并发抽选导致超卖：数据库层面加锁，确保库存扣减原子性；超卖时自动退款
